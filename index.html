Comandos de Git

Lista de comandos útiles para trabajar con Git

Comandos de Git

Comando Breve Descripción Parámetros y Descripción Ejemplo
git init Inicializa un nuevo repositorio Git en el directorio actual. - git init
git clone Clona un repositorio remoto en tu sistema local. repository_url: URL del repositorio remoto git clone https://github.com/ejemplo/repo.git
git add Agrega cambios al área de preparación (staging). file_path: Ruta del archivo a agregar git add archivo.txt
git commit Crea un nuevo commit con los cambios en el área de preparación. -m "mensaje": Mensaje descriptivo del commit git commit -m "Agrega cambios importantes"
git status Muestra el estado actual de los archivos en el repositorio. - git status
git pull Obtiene los cambios del repositorio remoto y los fusiona con tu rama local. - git pull
git push Sube los cambios locales al repositorio remoto. remote: Nombre del repositorio remoto git push origin master
git branch Lista todas las ramas en el repositorio. - git branch
git checkout Cambia a una rama existente o crea una nueva. branch_name: Nombre de la rama a cambiar o crear git checkout nueva_rama
git merge Fusiona una rama con la rama actual. branch_name: Nombre de la rama a fusionar git merge otra_rama
git log Muestra el historial de commits. - git log
git diff Muestra las diferencias entre commits o archivos. commit_hash: Hash del commit o file_path: Ruta del archivo git diff commit1 commit2
git reset Desanexa commits o restablece cambios en el área de preparación. commit_hash: Hash del commit o file_path: Ruta del archivo git reset HEAD archivo.txt
git remote Muestra los repositorios remotos configurados. - git remote -v
git fetch Obtiene cambios del repositorio remoto sin fusionarlos. remote: Nombre del repositorio remoto git fetch origin
git rebase Reorganiza commits en una rama. branch_name: Rama a la que se rebasa git rebase otra_rama
git tag Crea, lista o verifica etiquetas. - git tag -a v1.0 -m "Versión 1.0"
git cherry-pick Aplica un commit específico a una rama. commit_hash: Hash del commit git cherry-pick abc123
git stash Guarda cambios temporales sin hacer commit. - git stash save "Cambios temporales"
git remote add Agrega un repositorio remoto. remote_name: Nombre del repositorio remoto, repository_url: URL del repositorio remoto git remote add upstream https://github.com/upstream/repo.git
git remote remove Elimina un repositorio remoto. remote_name: Nombre del repositorio remoto git remote remove upstream
git clean Elimina archivos no rastreados y directorios. -n: Mostrar lo que se eliminaría, -f: Eliminar forzosamente git clean -n
git log --graph Muestra el historial de commits en un formato gráfico. - git log --graph

Conclusión

Los comandos de Git son herramientas esenciales para el control de versiones y la colaboración en proyectos de desarrollo de software. Esta tabla proporciona una visión general de algunos de los comandos más comunes utilizados en Git, junto con sus descripciones y ejemplos de uso. Es importante entender cómo funcionan estos comandos y cómo se pueden utilizar de manera efectiva para gestionar un repositorio Git y colaborar con otros desarrolladores. La práctica constante y la comprensión de los conceptos subyacentes son clave para aprovechar al máximo Git en tus proyectos.
